WHAT IS .GITIGNORE?
===================

The .gitignore file tells Git which files and folders to IGNORE (not track) in your repository.


WHY DO WE NEED IT?
==================

WITHOUT .gitignore (What Was Happening):
- git add -A tries to add EVERYTHING
- Includes bin/ folder (build outputs)
- Includes obj/ folder (compiler cache)
- Includes .vs/ folder (IDE settings)
- Result: "Filename too long" error!
- Repository becomes HUGE with useless files


WITH .gitignore (Now):
- git add -A adds everything EXCEPT what's in .gitignore
- Skips bin/ (can rebuild anytime)
- Skips obj/ (temporary files)
- Skips .vs/ (IDE specific)
- Result: Only source code & docs are added
- Repository stays clean and small


WHAT DOES OUR .gitignore EXCLUDE?
==================================

[Bb]in/              = Build output (DLLs, EXEs) - 100s MB
[Oo]bj/              = Compiler cache files - 100s MB
.vs/                 = Visual Studio settings - 10s MB
*.user               = Project user settings
.env                 = Environment variables (PASSWORDS!)
Migrations/          = Database migration scripts


SIMPLE RULE
===========

COMMIT:     Source code (.cs files, .csproj files)
COMMIT:     Documentation (README, markdown files)
COMMIT:     .gitignore itself

DON'T COMMIT:   Anything in bin/ or obj/
DON'T COMMIT:   IDE settings
DON'T COMMIT:   Build outputs
DON'T COMMIT:   Passwords or secrets


EXAMPLE
=======

Your Project:
MyApp/
  src/
    Program.cs          ✓ Commit this (source code)
  bin/Debug/
    MyApp.dll           ✗ Ignore this (build output)
  obj/
    cache files         ✗ Ignore this (generated)
  .gitignore            ✓ Commit this file

Result: Only src/ and .gitignore get pushed to git


THE PROBLEM YOU HAD
===================

git add -A tried to add this long filename:
src/.../obj/.../Employee.Infrastructure.GeneratedMSBuildEditorConfig.editorconfig

Windows has a path length limit - file path was TOO LONG

WITH .gitignore: That file is ignored, problem GONE!


WHAT HAPPENS NOW
================

1. You run: git add -A
2. Git sees .gitignore
3. Git automatically SKIPS bin/ and obj/ folders
4. Only your source code and docs are staged
5. No more "filename too long" errors
6. Push works perfectly!


KEY POINTS
==========

- .gitignore prevents tracking of generated/temporary files
- Makes repository smaller and faster
- Protects secrets (passwords, API keys)
- Each developer can regenerate bin/ and obj/ locally
- Every .NET project SHOULD have a .gitignore
